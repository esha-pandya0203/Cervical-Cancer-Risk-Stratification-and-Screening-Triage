## What this is

`api.py` is the Python inference backend that sits between the trained model bundle (`cervical_model_bundle_v2.joblib`) and the R Shiny frontend. You send it a patient's clinical features; it runs the full ensemble, scores confidence, generates explanations, renders three charts, and returns everything in one JSON response.

---

## Prerequisites

The bundle file must exist before the API will start. Run `02_model_v2.ipynb` to completion first — it writes `cervical_model_bundle_v2.joblib` to the project root.

Install the API dependencies on top of the modelling environment:

```bash
pip install fastapi uvicorn
```

Everything else (`joblib`, `scikit-learn`, `numpy`, `pandas`, `matplotlib`, `scipy`) is already required by the notebook.

---

## Running locally

From the project root (same directory as `api.py` and the bundle):

```bash
uvicorn api:app --host 0.0.0.0 --port 8001 --reload
```

The `--reload` flag restarts the server on file changes — useful during development, remove it in production.

Confirm it is up:

```bash
curl http://localhost:8001/health
```

Interactive docs are auto-generated by FastAPI and available at `http://localhost:8001/docs`.

## Endpoints

### `GET /health`

Liveness check. Returns bundle metadata so Shiny can confirm the backend is running and display the model version.

**Response fields:** `status`, `models` (list), `holdout_auc`, `holdout_prauc`, `threshold`, `bundle_path`.

---

### `GET /schema`

Returns the full input field specification — names, types, min/max ranges, required flags, and defaults. Shiny can call this on startup to build the input form dynamically rather than hard-coding field names.

---

### `POST /predict`

Main inference endpoint. Accepts a patient record, returns the complete DSS output.

**Request body (JSON):**

| Field | Type | Required | Notes |
|---|---|---|---|
| `age` | float | yes | Must be ≥ 10 |
| `sexual_partners` | float | yes | |
| `pregnancies` | float | yes | |
| `smokes_years` | float | no | Defaults to 0 |
| `hc_years` | float | no | Hormonal contraceptive years, defaults to 0 |
| `iud_years` | float | no | Defaults to 0 |
| `std_burden` | float | no | Sum of STD indicators, defaults to 0 |
| `cluster` | int (0–2) | no | Risk cluster. **Inferred automatically if omitted** using the saved KMeans model |

**Response fields:**

| Field | Type | Description |
|---|---|---|
| `prediction` | string | `"HIGH RISK"` or `"LOW RISK"` |
| `probability` | float | Ensemble P(Biopsy=1), 0–1 |
| `threshold` | float | Decision threshold used (tuned for ≤20% flag rate) |
| `confidence` | float | Confidence score, 0–1 |
| `confidence_tier` | string | Human-readable tier label |
| `model_probs` | dict | Per-model probability, keyed by model name |
| `ensemble_weights` | dict | OOF-optimised weight for each model |
| `conf_components` | dict | All six confidence formula terms with `weighted`, `raw`, and `description` per component |
| `nl_explanation` | string | Full natural language paragraph for the clinician |
| `warnings` | list | Clinical warning messages (empty list if none) |
| `scatter_png_b64` | string | PCA population scatter with patient marked — base64 PNG |
| `waterfall_png_b64` | string | Confidence component waterfall — base64 PNG |
| `contribution_png_b64` | string | Model contribution waterfall — base64 PNG |

**Minimal example (curl):**

```bash
curl -X POST http://localhost:8001/predict \
  -H "Content-Type: application/json" \
  -d '{
    "age": 32,
    "sexual_partners": 3,
    "pregnancies": 2,
    "smokes_years": 0,
    "hc_years": 4,
    "iud_years": 0,
    "std_burden": 1
  }'
```

**R Shiny integration:**

```r
library(httr2)

predict_patient <- function(patient_data) {
  request("http://localhost:8001/predict") |>
    req_body_json(patient_data) |>
    req_perform() |>
    resp_body_json()
}

# Render scatter plot from base64
output$scatter_plot <- renderImage({
  result  <- predict_patient(input_values())
  img_raw <- base64enc::base64decode(result$scatter_png_b64)
  tmpfile <- tempfile(fileext = ".png")
  writeBin(img_raw, tmpfile)
  list(src = tmpfile, contentType = "image/png", width = "100%")
})
```

## Confidence score formula

The confidence score combines six components:

```
score = a·x  +  b·y  −  c·z  +  d·m  +  e·(1−H)  +  f·agreement
```

| Term | Meaning |
|---|---|
| `x` | Cosine similarity to predicted class centroid |
| `y` | Ensemble probability (in the direction of the prediction) |
| `z` | Confusion risk — pull toward the opposite centroid |
| `m` | Demographic profile match (age proximity to typical positive case) |
| `1−H` | Certainty bonus — normalised binary entropy |
| `agreement` | Fraction of individual models agreeing with the ensemble decision |

Weights `(a, b, c, d, e, f)` are optimised on OOF predictions from the training set to maximise separation between correct and incorrect predictions.

**Confidence tiers:** ≥ 0.85 High · ≥ 0.70 Moderate · ≥ 0.55 Low · < 0.55 Very low

## Clinical warnings

The API automatically adds warning messages to the response in the following situations:

- **STD under-reporting:** `std_burden = 0` while other elevated risk factors are present and the prediction is HIGH RISK. STD under-reporting is documented in this dataset and the sensitivity analysis.
- **Low model agreement:** fewer than 60% of individual models agree with the ensemble decision.
- **IUD status unknown:** `iud_years = 0` for a HIGH RISK patient, flagging that the protective IUD signal may be absent due to missing data rather than confirmed non-use.

## Deploying beyond localhost

For the Shiny app hosted at `https://atorbati.shinyapps.io/DSS_Cervical/` to reach the API, the backend needs a public URL. The simplest options are:

**ngrok (quick demo):**
```bash
ngrok http 8001
# gives you a public https URL — update the Shiny request() call to point to it
```

**Render / Railway / Fly.io (persistent free tier):**
1. Add a `requirements.txt` listing `fastapi`, `uvicorn`, and all modelling dependencies.
2. Set the start command to `uvicorn api:app --host 0.0.0.0 --port $PORT`.
3. Upload the bundle file alongside `api.py` (or store it in a bucket and download on startup if it exceeds the platform's file size limit — the bundle is typically 50–200 MB).

**Docker:**
```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY api.py cervical_model_bundle_v2.joblib ./
CMD ["uvicorn", "api:app", "--host", "0.0.0.0", "--port", "8001"]
```

```bash
docker build -t dss-api .
docker run -p 8001:8001 dss-api
```

## Troubleshooting

| Symptom | Likely cause |
|---|---|
| `FileNotFoundError` on startup | Bundle not found — run `02_model_v2.ipynb` first, confirm `cervical_model_bundle_v2.joblib` is in the same directory as `api.py` |
| `422 Unprocessable Entity` | Invalid input — check field names and that required fields (`age`, `sexual_partners`, `pregnancies`) are present |
| `500 Internal Server Error` | Check uvicorn logs; most often a mismatch between the feature list in the bundle and the one hard-coded in `FEATURE_NAMES` |
| Charts are blank / tiny | Matplotlib backend issue — confirm `matplotlib.use("Agg")` is set before any other matplotlib import |
| TabPFN missing from `model_probs` | TabPFN was not licensed when the bundle was trained — the 4-model fallback ensemble is used instead, which is expected behaviour |
